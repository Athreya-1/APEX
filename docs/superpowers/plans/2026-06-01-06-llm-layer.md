# Plan 06 — LLM Layer & Model Routing (decision/parameter outputs)

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:test-driven-development`. The LLM **never writes a timeline** — it only emits **structured decisions/parameters** that the deterministic engine (Plans 02–05) consumes. All network-touching functions use **dependency injection** (`ModelCaller`) so they're unit-testable with a fake caller and require no API key. A pure, offline **local quick-add parser** provides a deterministic fallback and full coverage now.

**Goal:** A typed LLM layer: model routing (Haiku for cheap parse/classify, Sonnet for nuanced decompose/explain), zod schemas for every structured output, a JSON-extraction helper, an injectable `ModelCaller` with a real Anthropic implementation, a deterministic offline quick-add parser, and the four LLM functions (`parseQuickAdd`, `decomposeGoal`, `explainPlan`, `parseReplan`).

**New dependency:** `zod` (run `npm install zod` — pin whatever resolves).

**Architecture (all under `src/lib/llm/`):**
- `models.ts` — model id constants (env-overridable) + `routeModel(job)`.
- `schemas.ts` — zod schemas + inferred TS types for all structured outputs.
- `json.ts` — `extractJson(text)` (strip fences / find first JSON object) + `safeParse`.
- `quickAddLocal.ts` — pure deterministic NL parser (regex) → `ParsedTask | ClarificationRequest`.
- `client.ts` — `ModelCaller` type, `createAnthropicCaller()` (SDK wrapper, model routing).
- `functions.ts` — `parseQuickAdd`, `decomposeGoal`, `explainPlan`, `parseReplan` (each takes an injected `ModelCaller`, validates with schema, falls back where sensible).

### Task 2: Structured-output schemas (`schemas.ts`)

**Files:** Create `src/lib/llm/schemas.ts`; test `src/__tests__/llm-schemas.test.ts`.

**Spec (zod):**
```typescript
import { z } from 'zod'

export const TaskTypeTagSchema = z.enum(['lab','pset','reading','project','writeup','quiz','review','exam','other'])
export const CognitiveClassSchema = z.enum(['heavy_focus','light_admin','creative','physical','restorative'])

export const ParsedTaskSchema = z.object({
  kind: z.literal('task'),
  title: z.string().min(1),
  courseCode: z.string().nullable(),
  taskType: TaskTypeTagSchema,
  dueDate: z.string().nullable(),   // ISO or null
  doDate: z.string().nullable(),
  estimateHours: z.number().positive().nullable(),
  confidence: z.number().min(0).max(1),
})

export const ClarificationSchema = z.object({
  kind: z.literal('clarify'),
  question: z.string().min(1),
  missingFields: z.array(z.string()),
  partial: ParsedTaskSchema.partial().optional(),
})

export const QuickAddResultSchema = z.discriminatedUnion('kind', [ParsedTaskSchema, ClarificationSchema])

export const DecomposedHabitSchema = z.object({
  title: z.string().min(1),
  mode: z.enum(['time_blocked','check_off']),
  durationMins: z.number().int().positive(),
  frequencyType: z.enum(['daily','weekly','custom']),
  frequencyTarget: z.number().int().positive(),
  cognitiveClass: CognitiveClassSchema,
  rationale: z.string(),
})
export const GoalDecompositionSchema = z.object({
  goalTitle: z.string().min(1),
  habits: z.array(DecomposedHabitSchema).min(1),
})

export const PlanExplanationSchema = z.object({
  summary: z.string().min(1),
  highlights: z.array(z.string()),
})

export const ReplanInstructionSchema = z.object({
  intent: z.enum(['move','resize','add','remove','swap','rebalance','unknown']),
  targetRef: z.string().nullable(),
  params: z.record(z.string(), z.union([z.string(), z.number()])),
  rawText: z.string(),
})

export type ParsedTask = z.infer<typeof ParsedTaskSchema>
export type Clarification = z.infer<typeof ClarificationSchema>
export type QuickAddResult = z.infer<typeof QuickAddResultSchema>
export type GoalDecomposition = z.infer<typeof GoalDecompositionSchema>
export type PlanExplanation = z.infer<typeof PlanExplanationSchema>
export type ReplanInstruction = z.infer<typeof ReplanInstructionSchema>
```

### Task 4: Deterministic offline quick-add parser (`quickAddLocal.ts`)

**Files:** Create `src/lib/llm/quickAddLocal.ts`; test `src/__tests__/llm-quickAddLocal.test.ts`.

**Spec:** `parseQuickAddLocal(input, opts): QuickAddResult` — pure regex parser, NO network. `opts = { now: string /*ISO, for determinism*/, knownCourses?: string[] }`.
- **Course:** `/<token>` (e.g. `/213`, `/15-213`) or a known-course substring match → `courseCode`.
- **Duration:** `/4h`, `4h`, `90m`, `2.5h`, `/90m` → `estimateHours` (minutes ÷ 60).
- **Task type:** keyword map (`lab|pset|p-set|reading|read|project|writeup|write-up|quiz|review|exam|midterm|final`) → `TaskTypeTag` (default `other`). `midterm/final → exam`.
- **Relative dates** (relative to `opts.now`, UTC): `today`, `tomorrow`, `next week` (+7d), `in N days`, weekday names (`monday`..`sunday` → next occurrence), `due <date>`. Result is an ISO at 23:59:00 UTC of that date → `dueDate`.
- **Title:** input with the parsed tokens stripped, trimmed.
- **Confidence:** start 0.5; +0.2 if course found, +0.15 if due found, +0.15 if explicit duration. Cap 1.
- **Clarify trigger:** if NO course found AND no known-course match AND the text references a course-like type (`lab|pset|quiz|exam`) → return a `clarify` result asking for the course, `missingFields:['courseCode']`, with `partial` containing what was parsed.
